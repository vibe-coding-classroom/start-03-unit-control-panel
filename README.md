針對 **`start-03-unit-control-panel` (UI 佈局：HTML 按鈕與 CSS 控制面板)** 單元，這是學員將所有前端知識轉化為「可操作產品」的關鍵里程碑。它要求學員具備 **介面架構 (UI Architecture)** 與 **人因設計 (Ergonomic Design)** 的意識，學習如何利用 CSS Grid 構建穩定的控制網格，透過偽類實作物理級的按壓回饋，並最終完成一個具備沉浸感的 FPV 疊加面板。

以下是在 **GitHub Classroom** 部署其作業 (Assignments) 的具體建議：

### 1. 範本倉庫 (Template Repo) 配置建議
UI 佈局作業的核心在於「結構穩定性與互動細節」，範本應包含佈局框架與視覺資產，建議包含：
*   **📂 `index.html`**：**語義化骨架**。預設包含一個 `control-pad` 容器與 `video-stream` 背景，讓學員專注於元素排位。
*   **📂 `style.css`**：**佈局核心區**。學員在此練習 `grid-template-areas` 與 `:active` 狀態。留存 `/* TODO: Implement 3x3 Grid here */` 註釋。
*   **📂 `README.md`**：**UI 設計與觸控分析錄**。提供模板要求學員紀錄：在不同手機螢幕下，我的按鈕是否依然維持在 44x44px 以上？為什麼我選擇這組邊距？
*   **📂 `assets/`**：**SVG 圖示庫**。提供一套精美的向量箭頭圖示 (Up, Down, Left, Right, Stop)，確保學員能快速建立高品質的視覺反饋。

### 2. 作業任務部署細節

#### 任務 1：Grid 佈局挑戰 (D-Pad Layout Lab)
*   **目標**：打造「旋轉不散架」的堅韌佈局。掌握 CSS Grid 的語義化區塊命名與非對稱網格。
*   **Classroom 部署建議**：
    *   **3x3 網格實作**：使用 `grid-template-areas` 賦予按鈕特定的網格座標（如：`. up .`, `left stop right`）。
    *   **響應式驗證**：要求學員確保在手機橫向與直向模式下，十字鍵的相對位置與比例保持一致。
    *   **驗證要點**：**佈局健壯性審核**。學員是否使用了語義化的網格命名？按鈕在極端螢幕比例下是否會產生重疊或過度拉伸？

#### 任務 2：觸控區域與物理回饋優化 (Touch Feedback Lab)
*   **目標**：提升「操作手感」。應用 Fitts's Law 確保熱區足夠大，並透過動畫模擬按鈕被按下去的物理下沉感。
*   **Classroom 部署建議**：
    *   **熱區擴大實作**：確保每個按鈕的可點擊面積不小於 44x44px（即便視覺上看起來小一點）。
    *   **互動動畫實作**：利用 `transform: scale(0.95)` 與陰影變化實作 `:active` 狀態。設計一個明顯的 `:disabled` 樣式用於模擬斷線。
    *   **驗證要點**：在手機上操作時，是否有「我有按到」的明確視覺回饋？按鈕間距是否足夠防止誤觸？

#### 任務 3：沉浸式 FPV 疊加面板支援 (FPV HUD Overlay Lab)
*   **目標**：掌握圖層管理與視覺整合。打造如同專業無人機遙控器的抬頭顯示器 (HUD) 介面。
*   **Classroom 部署建議**：
    *   **圖層疊加實作**：利用 `position: absolute` 與 `z-index` 將控制層浮於影像層之上。
    *   **視覺風格整合**：實作半透明背景 (`glassmorphism`) 與統一的顏色編碼（如：停止鍵固定為紅色脈衝）。
    *   **驗證要點**：介面是否能清晰地疊加在背景之上且不遮擋核心視野？SVG 圖示的填充色是否會隨按壓狀態產生變化？

### 3. 排版美學與交互精準點評標準 (Aesthetics & Interaction Standards)
此單元的價值在於 **「對介面幾何結構的掌控力與對使用者肌肉記憶的尊重力」**：
*   [ ] **佈局邏輯純度**：CSS 程式碼是否乾淨地使用了 Grid，而非依賴大量的 Margin 補針？
*   [ ] **觸控人因合規性**：按鈕是否存在於拇指自然的掃描範圍內？點擊熱區是否符合移動端建議最小標準？
*   [ ] **視覺完整度**：當按鈕被連續快速點擊時，動畫與狀態切換是否流暢、不產生閃爍或跳動？

## 📝 UI 設計與觸控分析錄 (Student Submission)

*請在完成作業後填寫以下內容：*

### 1. 響應式佈局驗證
- **測試設備**：iPhone 13 / Chrome DevTools Mobile Mode
- **按鈕尺寸**：我的按鈕在該設備下是否維持在 44x44px 以上？ 是
- **分析**：我選擇了 12px 的 Gap 與 40px 的外邊距。這確保了在大螢幕下有足夠的呼吸空間，而在手機小螢幕下，按鈕依然保持在 50px-60px 之間，符合人因工程的最小點擊熱區標準。

### 2. 觸控人因設計
- **操作手感**：我使用了 `transform: scale(0.92)` 配合 `box-shadow` 的內縮與發光效果。當用戶按下按鈕時，視覺上的「下沉感」與顏色從半透明變為亮藍色的切換，提供了極佳的物理回饋。
- **防誤觸設計**：透過 CSS Grid 的精確定位，我將十字鍵排列成標準的 3x3 佈局，並在按鈕之間保留了 12px 的間隔 (Gap)，防止手指在大範圍移動時同時觸發兩個方向。

### 3. FPV 視覺整合
- **圖層管理**：HUD 疊加層使用了 `position: absolute` 定位，並透過 `z-index: 10` 確保其始終浮在影片背景層 (`z-index: 1`) 之上。
- **美學選擇**：我選擇了深色半透明的 Glassmorphism 風格，配合 `backdrop-filter: blur(20px)`。這能確保控制面板在多變的背景畫面下依然清晰可見，同時不完全遮擋後方的飛行視角。
